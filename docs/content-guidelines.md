# 內容撰寫規範

## 家長版文字

- 以家長生活情境開頭，例如「孩子今天不能到校怎麼辦？」。
- 優先回答家長最需要知道的下一步。
- 避免行政術語堆疊；必要術語需用白話補充。
- 涉及時間、地點、申請流程或表單時，必須標示來源狀態與查核日期。
- 未查核內容只能寫成「待確認」或「需處室確認」，不得寫成正式規則。

## 學生版文字

- 使用短句與「我可以怎麼做」語氣。
- 說清楚可以找誰，例如家長、導師、學務處、輔導室、健康中心或信任的大人。
- 不要求學生自行判斷事件性質。
- 涉及安全、健康、人際、霸凌、性平或申訴時，文字只做求助導向。

## 敏感議題寫法

- 霸凌與性平內容不得寫成認定教學。
- 不讓家長或學生自行判斷事件是否成立。
- 不提供法律結論。
- 不蒐集事件細節。
- 不呈現個案資料。
- 前台文字應強調求助管道、學校依法處理、保護當事人與不揭露個資。

## 法規查核

- 法規摘要不得未查核即上線。
- 涉及霸凌、性平、學生申訴、獎懲、輔導管教與個資等內容，需由權責處室或具相關專業人員協助查核。
- 法規連結、查核日期與查核角色應納入資料欄位或維護紀錄。

## 校網資料引用

- 校網資料可作為來源連結或摘要依據。
- 不直接複製未查核的公告全文。
- 舊年度資料需確認仍有效才能使用。
- 若 URL 不確定，先留空並標示需查核，不自行補上不確定連結。

## 個資禁止

- 不放學生姓名、家長姓名、班級座號、個人手機或私人聯絡方式。
- 不放健康個資。
- 不放事件細節或案件描述。
- 不在公開頁面蒐集使用者輸入資料。

## token / secret 禁止

- token 與 secret 只能放在正式部署平台的安全設定中。
- 不寫入 repo、文件、頁面、資料檔或紀錄。
- 第一版靜態 MVP 不建立任何 secret 或 token。

## 連結檢查

- 每個外部連結上線前需確認有效。
- 每次內容更新需檢查重要表單與法規連結。
- 若連結失效，卡片應標示待更新，不應保留為正式流程依據。
