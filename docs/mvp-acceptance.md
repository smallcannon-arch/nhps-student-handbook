# MVP 驗收標準

第一版靜態 MVP 驗收重點，是確認頁面可以被家長快速理解、資料格式安全、敏感議題不越界，且後續可進入內容查核。

## 前台可用性

- 手機寬度 390px 無橫向溢出。
- 校園生活分類清楚可見。
- 搜尋常用詞可命中相關卡片。
- 常用卡片可顯示標題、致家長、孩子可以這樣做、聯繫窗口、來源狀態與更新日期。
- 字級清楚，對比足夠，手機閱讀不擁擠。

## 資料完整性

- 每張卡片有 `owner`、`reviewer`、`updated_at`、`source_status`。
- 每張卡片可追蹤來源狀態。
- 未查核內容明確標示需處室確認。
- 不直接複製未查核公告全文。

## 安全邊界

- 無學生個資。
- 無健康個資。
- 無事件個案資料。
- 無 token 或 secret。
- 敏感議題不做個案判斷。
- 霸凌與性平內容只提供求助導向，不做認定教學或法律結論。

## 技術邊界

- 不依賴外部 API。
- 不依賴外部 CDN。
- 不使用 npm 或 build tool。
- GitHub Pages 可正常開啟靜態頁。
- 不需要 Worker deploy。
- 不建立 Google Sheets、GAS、Cloudflare Worker、LINE Bot 或 LINE Rich Menu。
