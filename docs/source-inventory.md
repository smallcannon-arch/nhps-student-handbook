# 校網公開資料盤點摘要

本文件記錄第一版學生手冊可參考的公開來源類型與查核原則。公開資料可作為入口與來源線索，但不代表可直接複製或立即定稿；涉及流程、時間、處理原則或敏感議題時，仍需權責處室確認。

## 已找到的公開來源

- 校網首頁。
- 行事曆或活動公告入口。
- 重大議題專區。
- 反霸凌專區。
- 性別平等教育專區。
- 處室簡介與處室職掌。
- 校園午餐入口。
- 交通安全相關入口。

## 需處室確認的資料

- 日常上學時間、放學時間與特殊作息。
- 路隊、家長接送、安親班接送與雨天放學安排。
- 請假、遲到、早退與出缺席流程。
- 健康中心服務、學生受傷、發燒、服藥與通知流程。
- 午餐訂退餐、特殊飲食、餐具、飲水與費用規則。
- 行動載具與智慧手錶規範。
- 失物招領流程。
- 學生申訴與輔導資源。

## 需人工提供的資料

- 各處室確認過的最新流程文字。
- 常用表單的最新版與有效下載位置。
- 家長接送區、路隊位置與校園安全注意事項。
- 高年級學生友善文字。
- 敏感議題的求助導向文字與查核日期。

## 不建議直接納入的資料

- 舊年度公告或未確認仍有效的 PDF。
- 未查核的時間、地點、流程與規則。
- 含有個人資料、個別事件或可識別人物的內容。
- 霸凌或性平相關資料中的個案描述。
- 需要法律或事件成立判斷的文字。

## 來源追蹤欄位原則

每張卡片應保留：

- `source_urls`：公開來源連結，可空白但需說明狀態。
- `source_status`：已找到、需處室確認、需人工提供或不建議直接使用。
- `source_type`：校網公開入口、人工查核、處室文件或其他。
- `source_checked_at`：來源查核日期。
- `needs_office_review`：是否需處室確認。
- `source_owner_note`：來源與維護提醒。

## 五、圖卡處理原則

1. PDF 內原圖、截圖、表格不直接公開使用。
2. 未來採「文字卡 + 雲端整理圖卡連結」。
3. 圖卡需依「學生手冊視覺風格 V1」重製。
4. 前台首頁與列表不直接顯示大量圖卡。
5. 圖卡只在卡片按鈕中連結。
6. 圖卡欄位日後可規劃：
   - `visual_card_title`
   - `visual_card_url`
   - `visual_card_status`
   - `visual_card_type`
   - `visual_card_note`
7. 現階段先不修改 JSON 欄位，等第一批圖卡需求確認後再處理。

- 教師導護輪值細節、交接方式、值勤提醒與內部工作分派。
- 教師常用網、需登入入口、內部短網址、Google Workspace 入口或共享資料夾路徑。
- LINE 群組記事本、LINE 相簿、私傳照片或內部群組提醒。
- 教師工作提醒、教師填寫紀錄、教師拍照回報與行政交辦語氣。
- 內部通報時限細節若未轉寫為家長友善求助內容。
- 學生名冊、學生代表名冊、榮譽兒童名冊或可識別個人的資料。
- 原始圖檔、截圖、表格、範例圖片或未重製圖卡。
- 涉及個資、案件、健康資料、家庭糾紛或內部行政運作的內容。
- 可能讓家長或學生自行判斷霸凌、性平、通報或法律責任成立與否的文字。

## 七、建議整合策略

建議先採 A：更新 `docs/source-inventory.md`，記錄 PDF 來源與拆解表。下一步再採 C：新增 5～8 張 draft 卡片。

不建議本輪直接新增全部卡片，原因如下：

- PDF 原始定位是教師聯繫事項，需轉寫後才能供家長與學生閱讀。
- 多數主題涉及作息、遲到、請假、接送、路隊、通報與學生安全，需學務處／生教組確認。
- 部分內容包含教師內部工作提醒、內部入口、LINE 群組與名冊，不適合前台公開。
- 圖卡需求尚未確認，現階段不新增 `visual_card_*` 欄位，避免資料結構過早擴張。
- 原 PDF 與原圖不公開，後續若需要圖卡，應依學生手冊視覺風格重製。

暫不處理項目：

- 暫不直接新增全部卡片。
- 暫不新增 `visual_card_*` 欄位。
- 暫不公開原 PDF。
- 暫不直接使用 PDF 原圖。
- 暫不把教師常用網、內部資料夾、LINE 群組或名冊放入前台。

# 第一批 10 張學生手冊圖卡盤點

本章用於規劃第一批學生手冊懶人包圖卡。圖卡未來放在雲端整理圖卡資料夾，不直接塞滿前台；前台仍以文字卡為主，圖卡作為延伸說明連結。原始 PDF 圖、截圖、表格不直接使用，圖卡需依「學生手冊視覺風格 V1」重製。現階段只做盤點，不建立圖檔、不新增 JSON 欄位、不改前台。

## 一、圖卡使用策略

1. 前台首頁與搜尋結果不直接顯示大量圖卡。
2. 每張文字卡未來可選擇性顯示「查看圖卡」按鈕。
3. 只有圖卡狀態為「已發布」且連結可公開時，前台才顯示按鈕。
4. 圖卡放在雲端整理圖卡資料夾，原始素材與審稿圖不公開。
5. 原始承辦資料、教師內部資料、需登入連結不得作為前台圖卡連結。
6. 圖卡只作輔助理解，正式資訊仍以文字卡與處室查核後內容為主。

## 四、統一視覺風格

「學生手冊視覺風格 V1」建議如下：

- 現代教育行政風。
- 深藍、米白、霧藍、暖灰、低彩度綠與低彩度橘。
- 扁平插畫。
- 乾淨留白。
- 不商業。
- 不幼兒園卡通。
- 不小紅書風。
- 不放文字、數字、Logo、QR Code。
- 不畫寫實人臉。
- 不呈現衝突、侵犯、哭泣特寫或個案情節。

## 五、圖卡雲端管理建議

建議未來規劃雲端資料夾如下；本輪不建立資料夾：

```text
學生手冊圖卡/
  01_原始資料_內部/
  02_圖卡審稿/
  03_公開圖卡/
```

- `01_原始資料_內部` 不公開。
- `02_圖卡審稿` 不給前台連結。
- `03_公開圖卡` 才可作為前台「查看圖卡」連結。
- 前台只連結整理後圖卡，不連原始 PDF 或內部資料。

## 七、排除原則

圖卡不得包含：

- 原始 PDF 截圖。
- 教師內部連結。
- LINE 群組資訊。
- 學生或家長個資。
- 真實姓名、班級座號、個人手機。
- 健康個資。
- 霸凌／性平個案情節。
- 未查核的正式流程或期限。
- 可辨識真實學生或教師臉孔。
- 校內敏感空間細節。
